# 「易见」产品策划方案 / 开发说明书（PRD）

> 交付对象：Claude Code（开发执行）
> 文档目标：基于本文件可直接开始开发，无需再追问产品形态与逻辑。
> 形态：微信公众号 H5（移动端为主，PC 浏览器自适应可用）。

## 0. 给 Claude Code 的总体说明

这是一个把《周易》起卦解卦逻辑包装成"决策辅助 / 情绪梳理工具"的轻量 Web 应用。

请注意三条贯穿全程的硬约束：

1. **合规优先**：前端任何文案、AI 输出，都不得出现"算命、占卜、预测吉凶、转运、保平安、必然、注定、血光、破财"等词。一切表述用"提示 / 参考 / 可能 / 思路梳理"。详见第 11 节。
2. **算法是后台核心**：起卦/成卦/变卦/互卦逻辑必须严格正确，详见第 4 节。时间起卦部分会有现成代码（开发方提供），按第 4.1 节的接口对接即可。
3. **支付与账号涉及真实资质**：微信公众号、微信支付商户号、短信服务等账号注册由开发方（人类）完成，Claude Code 只负责对接，不要尝试注册账号或填写敏感凭证，凭证一律通过环境变量注入。

### 2.1 包装原则：前端语言 ↔ 后台逻辑对照

### 2.2 合规红线（开发与文案必须遵守）

- 结果页、设置页常驻免责声明；用户首次进入弹窗确认一次。

## 4. 核心算法：起卦引擎

> 引擎应封装为独立模块，输入"起卦参数"，输出"卦象对象"。前端不参与任何卦象计算。

### 4.0 基础数据：八卦

先天八卦数与爻象（爻自下而上，1=阳爻，0=阴爻）：

| 卦名 | 卦符 | 先天数 | 爻象[初,中,上] |
|---|---|---|---|
| 乾 | ☰ | 1 | [1,1,1] |
| 兑 | ☱ | 2 | [1,1,0] |
| 离 | ☲ | 3 | [1,0,1] |
| 震 | ☳ | 4 | [1,0,0] |
| 巽 | ☴ | 5 | [0,1,1] |
| 坎 | ☵ | 6 | [0,1,0] |
| 艮 | ☶ | 7 | [0,0,1] |
| 坤 | ☷ | 8 | [0,0,0] |

一个六爻卦 = 下卦（初/二/三爻）+ 上卦（四/五/六爻）。爻位 1–6 自下而上。

### 4.1 时间起卦（参考下方代码）

开发方会提供现成的时间起卦代码。Claude Code 只需把它封装成统一接口：

- 输入：起卦时刻（默认取服务器当前时间，或用户指定）
- 内部：按梅花易数年月日时法（农历地支数）计算上卦数、下卦数、动爻
- 输出：见 4.4 的统一卦象对象

若现有代码只产出"上卦数 / 下卦数 / 动爻"，则成卦/变卦/互卦统一交给 4.3 处理。

### 4.2 三数起卦（需新开发）

用户依直觉报三个数字 a、b、c（输入校验：正整数，建议限制 1–99，至少 1 个非零）。

```
取八卦数:  r % 8, 若结果为 0 则取 8
上卦数 = a % 8  (0→8)
下卦数 = b % 8  (0→8)
动爻   = (a + b + c) % 6  (0→6)   // 1~6，自下而上
```

示例：a=15, b=8, c=23 → 上卦 15%8=7（艮）；下卦 8%8=0→8（坤）；动爻 (15+8+23)%6=46%6=4 → 第 4 爻。

### 4.3 成卦 / 变卦 / 互卦（统一逻辑）

输入：上卦数、下卦数、动爻（1–6）。

```
本卦六爻(自下而上) = 下卦爻象[3位] + 上卦爻象[3位]
变卦 = 复制本卦六爻，将第「动爻」位的爻翻转 (1↔0)
互卦 = 下互(本卦第2,3,4爻) + 上互(本卦第3,4,5爻)
```

由六爻还原"上卦/下卦卦名"：取下三爻匹配八卦爻象表得下卦，上三爻得上卦。再用 (上卦,下卦) 查 64 卦表得卦名等信息。变卦、互卦同样查表。

### 4.4 统一输出：卦象对象

```json
{
  "method": "time | number",
  "input": { "datetime": "...", "numbers": [a,b,c] },
  "dongYao": 4,
```

## 6. AI 解卦模块

### 6.1 调用时机（控制成本）

- **免费预览**：起卦后立即生成，内容 = 卦名 + 卦象图 + `imageText` 一句意象 + 一句简短提示。这部分尽量基于卦数据 + 轻量模型，token 消耗低。
- **完整解读**：仅在用户**支付成功后**调用大模型生成，生成结果存库（`divinations.ai_reading`），之后查看不再重复扣费。

### 6.2 解读输出固定结构（前端按此渲染）

```json
{
  "imagery": "卦象意象，1句",
  "hints": ["卦象给你的提示，3-4句"],
  "reflections": ["可以想一想，2-3个反思问题"],
  "direction": "一个值得考虑的方向，温和建议，1-2句",
  "bianHint": "结合变卦/互卦的'另一面/后续参考'，1-2句"
}
```

### 6.3 大模型 System Prompt（直接使用，可微调）

```
你是一个《周易》文化研习的辅助工具。用户会给出一个正在纠结的现实问题，
以及一卦的卦象信息（本卦、变卦、互卦、动爻、卦辞爻辞）。

你的任务：从周易卦象的角度，帮用户「梳理思路、换个角度看问题」。

严格遵守：
1. 你不是算命师，不预测未来，不下吉凶结论。
2. 禁止使用：算命、占卜、预测、注定、必然、一定、凶、大凶、血光、
   破财、转运、化解、保平安 等词汇。
3. 全程用"提示、可能、或许、值得留意、不妨想想"等中性措辞。
4. 不对医疗、投资、婚姻、法律等给出指令性结论，只提供思考角度。
5. 语气温和、像一个冷静的朋友，不制造焦虑。
6. 必须严格输出指定 JSON 结构，不要多余文字。

输出 JSON 字段：imagery, hints(数组), reflections(数组), direction, bianHint。
reflections 必须是开放式反思问题，引导用户自己找答案。
```

User message 拼接：用户问题、主题、心情标签、本卦/变卦/互卦卦名与卦辞爻辞、动爻。

### 6.4 兜底

模型超时或返回不合规（命中禁词）→ 重试一次 → 仍失败则返回基于卦数据的模板化解读，并记录日志。

## 7. 页面与路由设计

SPA 路由，移动优先。所有页面底部常驻一行小字："周易文化研习工具 · 仅供学习与娱乐参考"。

### 7.1 路由表

| 路由 | 页面 | 说明 |
|---|---|---|
| `/` | 首页 | 入口 |
| `/ask` | 提问页 | 输入困惑 |
| `/cast` | 起卦页 | 选择起卦方式 + 成卦动画 |
| `/result/:id` | 结果页 | 卦象 + 解读（免费预览/完整）|
| `/pay/:orderId` | 支付页 | 微信内 JSAPI / PC 扫码 |
| `/history` | 卦象记录 | 历史列表 |
| `/me` | 我的 | 会员状态、设置、免责声明 |
| `/login` | 登录 | 仅 PC/微信外手机号登录用 |

### 7.2 ① 首页 `/`

- 视觉：新中式/治愈系，留白、淡墨、莫兰迪低饱和色。**禁止**红黑、香烛、大师头像等算命风。
- slogan："拿不准的时候，来取一卦，换个角度想想。"
- 主按钮："说说你正在纠结的事" → `/ask`
- 次入口："今日一卦"、"我的卦象记录"
- 首次进入弹一次免责声明确认弹窗。

### 7.3 ② 提问页 `/ask`

- 引导语："把你正纠结的事说出来，越具体越好。例：要不要接这个 offer？"
- 多行输入框（必填，建议 5–200 字，前端校验）。
- 主题标签（单选，必填）：事业 / 感情 / 选择 / 人际 / 其他。
- 心情标签（可选）：焦虑 / 迷茫 / 期待 / 平静。
- 按钮："下一步，取一卦" → `/cast`，携带 question/topic/mood。

### 7.4 ③ 起卦页 `/cast`

两张选择卡片：

- 「此刻起卦」→ method=time。文案："你想到这个问题的当下，本身就带着线索。"
- 「报数起卦」→ method=number。文案："心里默念问题，跟着直觉报三个数字。"
  - 报数交互：三个数字选择器或滚轮，增加仪式感。
- 选定后调用 `POST /api/divination/cast`，期间播放 2–3 秒成卦动画（六爻自下而上逐根浮现）→ 成功后跳 `/result/:id`。

### 7.5 ④ 结果页 `/result/:id`（核心页）

固定模块，渲染顺序：

1. **卦象**：本卦名 + 卦象图（六爻图形）+ 一句意象（`imagery`）。不显示吉凶判语。
2. **卦象给你的提示**：`hints`，3–4 句。
3. **可以想一想**：`reflections` 反思问题。
4. **一个值得考虑的方向**：`direction`。
5. **事情的另一面**：`bianHint`（基于变卦/互卦）。
6. 底部：免责声明 + 操作区（保存 / 再问一个 / 分享）。

**付费门控**：未付费用户，模块 1 与 hints 第 1 条免费可见；模块 2(剩余)–5 打码，叠加付费墙："完整解读已生成，解锁查看周易给你的详细梳理" + 价格选项 → 进入支付。已付费/会员则全部展开。

**分享**：生成一张精美卦象卡片图（卦名+意象+二维码），用于公众号内传播拉新。

### 7.6 ⑤ 我的 `/me`

会员状态（类型、到期日 / 剩余次数）、卦象记录入口、"最近常问的主题"轻洞察、免责声明、联系方式、退出登录。

## 9. 登录与用户体系

### 9.1 微信内（主路径）

1. 进入站点检测在微信内 → 重定向至微信网页授权 `snsapi_base` 静默授权。
2. 拿 `code` → 后端换 `openid`（必要时 `snsapi_userinfo` 取头像昵称）。
3. 按 openid 查/建用户，发本站登录态（JWT 或 session）。

### 9.2 PC / 微信外

- `/login` 手机号 + 短信验证码登录（接入阿里云/腾讯云短信）。
- 验证码存 Redis，60 秒重发限制，5 分钟有效。

> 同一用户可能既有 openid 又有 phone，按手机号或 unionid 做账号合并（阶段二可做，MVP 可暂不合并）。

### 10.2 支付流程

**微信内 JSAPI**：`POST /api/order/create` → 后端微信支付下单（JSAPI，传 openid）→ 返回前端调起参数 → `WeixinJSBridge`/`wx.chooseWXPay` 调起 → 微信异步回调 `notify_url` → 后端验签、改订单状态、发放权益。

**PC Native 扫码**：`POST /api/order/create`（trade_type=NATIVE）→ 后端拿 `code_url` → 前端生成二维码 → 用户手机扫码支付 → `notify_url` 回调发权益 → 前端轮询 `GET /api/order/:id/status` 刷新。

### 10.3 权益发放（在支付回调里，幂等处理）

- single → 对应 `divination.is_unlocked = 1`，并触发完整 AI 解读生成写入 `ai_reading`。
- card5 → `remaining_credits += 5`。
- monthly → `member_type='monthly'`，`member_expire_at` 顺延 30 天。

> 回调必须做幂等（同一 `transaction_id` 只发一次权益）。商户号经营类目按"文化娱乐/软件服务"等申请，不得报"算命占卜"。

## 14. 开发里程碑与优先级

| 阶段 | 内容 | 产出 |
|---|---|---|
| M1 | 起卦引擎 + 64 卦数据 + 三数起卦 + 成卦/变卦/互卦 | 引擎模块 + 单元测试 |

## 15. 交付给 Claude Code 的注意事项

1. 起卦引擎要有充分单元测试：用已知输入（如第 4.2 节示例）校验上卦/下卦/动爻/变卦/互卦正确。
2. 微信公众号 AppID、商户号、API 证书、短信与大模型 key 全部走环境变量，**不要写进代码、不要尝试注册任何账号**。
3. 微信网页授权与支付要求 HTTPS 且域名已备案——本地开发用内网穿透联调，部署由开发方配置。
4. 支付回调必须验签 + 幂等。
5. AI 解读：免费预览轻量、完整解读付费后生成并落库，避免无效 token 消耗。
6. 全程执行第 11 节合规规范，禁词过滤覆盖 AI 输出。
7. 时间起卦沿用开发方提供的现成代码，按第 4.1 节接口封装；三数起卦与成卦/变卦/互卦为新开发。
