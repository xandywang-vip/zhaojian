# 「易见」产品策划方案 · v2（终版）

> 本文档整合了三件事：
> ① 保留产品**初心**——为"犹豫不决、拿不准"的人提供易经起卦解卦的决策梳理工具，并以此变现（按次 / 包月）；
> ② 保留**原有技术决策**——解卦用 LLM 生成，**不用 64 卦人工模板**；公众号 H5 形态（移动优先 + PC 自适应）；
> ③ 吸收「照见」PRD 的优点——疗愈化语言、六段式卡片结构、共时性定位、心境墙、安全声明；
> 同时**修正「照见」PRD 的问题**——补齐危机干预、收回"心理咨询"专业身份主张、补 AI 标识与防沉迷、保留变现、修正隐私措辞。
>
> **本文档取代此前的《易见-产品策划方案.md》与《易见-AI解卦Prompt.md》，是交付给 Claude Code 的唯一主文档。**

---

## 0. 给 Claude Code 的总体说明

「易见」是一个把《周易》起卦解卦逻辑，包装成"决策梳理 + 情绪觉察"轻量 Web 工具的产品。用户在犹豫不决时写下困扰，起一卦，由 LLM 从卦象角度生成一段帮他**理清这个决定、安顿情绪**的解读。

四条贯穿全程的硬约束：

1. **合规优先**：见第 2、3 节。两条红线——不做算命占卜、不自称心理咨询/治疗。
2. **安全优先**：用户会在深夜、情绪低谷时使用本产品。第 3 节的"危机内容识别与干预"是上线前的必做项，不是可选项。
3. **算法正确**：起卦/成卦/变卦/互卦逻辑必须严格正确，见第 5 节。
4. **凭证安全**：公众号、微信支付、短信、大模型等账号注册由人类完成，Claude Code 只负责对接，凭证一律走环境变量，**不要尝试注册账号或填写敏感凭证**。

## 1. 产品概述

| 项 | 内容 |
|---|---|
| 产品名（对外）| 易见 |
| 产品名（公众号注册名）| 朴素学习类名称，如"周易文化研习"，避免营销词 |
| Slogan | 拿不准的时候，换个角度，照见自己 |
| 产品形态 | 微信公众号 H5 单页应用，移动优先 + PC 自适应 |
| 目标用户 | 18-35 岁城市白领、学生、自由职业者，面临情感、职业、选择、人际等成长性困扰 |
| 核心价值 | 用《周易》卦象作为"思考的镜子"，帮用户理清当下的纠结、安顿情绪，**自己把决定想清楚** |
| 商业模式 | 免费起卦+预览做钩子；完整解读单次解锁 / 次卡 / 包月会员 |

> 关于命名：沿用原决策的「易见」（含"易"，谐音"遇见"，意为"更容易看清"）。不采用「照见」——该词出自《心经》，与易经产品混搭会带来宗教属性上的定位含糊。

### 1.1 产品是什么 / 不是什么

- **是**：一个传统文化研习 + 决策梳理 + 情绪觉察的工具，提供"从卦象角度的参考解读"。
- **不是**：① 不是算命、占卜、运势预测工具；② 不是心理咨询 / 心理治疗服务，不替代专业帮助。
- 不提供任何"转运、化解、开光、法事、改运"功能或商品。

### 1.2 初心与「照见」PRD 的关键分歧（务必理解）

「照见」PRD 把产品改成了**纯情绪安抚**工具，明确"不给出指令性结论"。这丢掉了产品初心——用户来，是因为**拿不准、要做决定**。

本版的取舍：**保留"决策梳理"这个核心效用，但用疗愈化的语言来承载它**。我们不替用户做决定（这是安全红线），但要通过"反思问题 + 可执行的小行动"，真正帮他把这个决定想清楚——而不是只说一句"允许自己停在这里"。这是"有用"和"安全"之间的平衡点。

### 1.3 典型用户场景

- **深夜纠结**：反复想要不要分手。写下令自己难受的瞬间，起一卦，读到"困住你的也许不是这段关系本身，而是'不敢承认答案'这件事"——情绪被接住，也被推着往前想了一步。
- **职业抉择**：两个 offer 难选。报三个数字起卦，读到"改变不一定因为旧的全坏，可能是你心里已经有了新标准"——多了一点确定感，并拿到三个帮自己梳理利弊的提问。
- **日常自我关怀**：通勤时打开心境墙，回看自己保存过的卡片和写下的关怀语，感受内在的连续性。

## 2. 定位与合规红线

### 2.1 包装原则：前端语言 ↔ 后台逻辑对照

后台逻辑不变，只改用户看到的字。

| 后台真实逻辑 | 前端展示语言 |
|---|---|
| 起卦 | 为这个困扰找一个可以看看的角度 |
| 占卜 / 算命 | 周易视角的解读 / 思路梳理 |
| 预测吉凶 / 算运势 | 此刻处境的提示 |
| 此卦为凶 | 卦象提示当前阻力较大，提醒你…… |
| 大师解卦 | 智能周易解读（AI 生成）|

### 2.2 两条合规红线

**红线一：不做算命占卜。** 全站文案与 AI 输出不出现"算命、占卜、预测、卜卦、运势、注定、必然、吉、凶、悔、吝、转运、改运、化解、开光、保平安"等词。微信公众号代码审核明确不允许算卜算命类内容，必须从定位到文案彻底规避。

**红线二：不自称心理咨询 / 心理治疗。** 这是对「照见」PRD 的重要修正——该版本反复使用"心理自助工具、心理咨询四步法、资深心理咨询师审校"。"心理咨询"在国内是有国家标准、受卫健与网信监管的术语，自称会把产品放进"按心理咨询/医疗标准被审视"的位置。

- 对外定位用"决策梳理 / 情绪觉察 / 传统文化研习工具"，**不用**"心理咨询""心理治疗""疗愈师"等词。
- 不在用户可见文案中宣称"由心理咨询师撰写/审校"。运营可在内部参考心理学方法做内容质控，但不作为对外卖点。
- 全程、显著、反复地呈现非治疗声明（见 11.1）。

### 2.3 AI 标识（因使用 LLM，必做）

本产品解读由大模型生成，需在显著位置持续标识，例如：

> 本产品的解读由人工智能生成，不具备人类情感意识，仅供参考，不能替代专业心理帮助。

放置：欢迎引导页、结果页解读区上方。

### 2.4 防沉迷 / 反复抽卡限制

反复起卦本身是焦虑驱动的强迫行为，既是合规要求也是产品善意：

- 同一条困扰，24 小时内只生成一次解读（再次提交相同/高度相似困扰，引导回看已有卡片）。
- 每位用户每日起卦上限 3 次，超出时温和提示："今天已经照见过几次了，给自己一点消化的时间。"
- 单次会话连续使用时长较长时（如超过 40 分钟），弹一次柔和提醒，建议先休息。

## 3. 安全设计：危机内容识别与干预【上线前必做】

> 这是「照见」PRD 缺失、而本产品**最重要**的一节。用户会在情绪低谷、深夜写下"令自己难受的事"，这正是自伤、自杀念头最易出现的场景。

### 3.1 触发时机

在用户**提交困扰描述、进入起卦之前**，对输入文本做一次危机内容检测。

### 3.2 检测方式

- 第一层：关键词 / 规则匹配（维护一份危机词库：涉及自伤、自杀、轻生、严重绝望、伤害他人等表达）。
- 第二层（推荐）：用一个轻量 LLM 分类调用判断"该文本是否包含心理危机信号"，弥补关键词漏判。
- 检测只作用于**用户输入**，不作用于卦辞原文。

### 3.3 命中后的处理

- **不进入起卦流程**，不生成卦象解读。
- 切换到一个温和的"关怀页面"，而非冷冰冰的报错。
- 关怀页面要做到：① 用温暖、不评判的语气表达看见与关心；② 明确告诉用户"易见只是一个梳理工具，此刻你值得更专业、更有力的帮助"；③ 直接呈现权威求助渠道。
- **内容安全铁律**：关怀页面、以及全站任何文案与 AI 输出，**绝不描述任何自伤/轻生的具体方式方法**，只往"寻求帮助、联系他人"的方向引导。

### 3.4 求助资源

关怀页面需内置官方权威心理援助渠道，例如全国统一心理援助热线、各地心理危机干预热线等。

> 注意：具体热线号码以**上线时官方公布的最新信息为准**，开发方务必在上线前逐一核实并定期复核，不要使用过期或未经核实的号码。

### 3.5 关怀页面参考文案

> 看到你写下的这些，我们很想先停下来，认真地对你说一句：你现在承受的，真的很不容易。
> 易见只是一个帮人换角度想问题的小工具，但此刻的你，值得比一卦更有力的陪伴。
> 如果愿意，可以联系下面的渠道，那一头是真实的、受过专业训练的人，他们会认真听你说。
> 你也可以告诉身边一个你还信任的人。你不需要一个人扛。
> [ 求助渠道列表 ]

## 5. 核心算法：起卦引擎

> 引擎封装为独立模块：输入起卦参数 → 输出统一卦象对象。前端不参与任何卦象计算。

### 5.1 基础数据：八卦

爻自下而上，1 = 阳爻，0 = 阴爻。

| 卦名 | 卦符 | 先天数 | 爻象[初,中,上] |
|---|---|---|---|
| 乾 | ☰ | 1 | [1,1,1] |
| 兑 | ☱ | 2 | [1,1,0] |
| 离 | ☲ | 3 | [1,0,1] |
| 震 | ☳ | 4 | [1,0,0] |
| 巽 | ☴ | 5 | [0,1,1] |
| 坎 | ☵ | 6 | [0,1,0] |
| 艮 | ☶ | 7 | [0,0,1] |
| 坤 | ☷ | 8 | [0,0,0] |

六爻卦 = 下卦（初/二/三爻）+ 上卦（四/五/六爻）。爻位 1–6 自下而上。

### 5.2 时间起卦（沿用现成代码）

开发方提供现成的时间起卦代码（梅花易数年月日时法）。Claude Code 封装成统一接口：输入起卦时刻（默认服务器当前时间），产出上卦数 / 下卦数 / 动爻，再交 5.4 统一处理。

### 5.3 三数起卦（新开发，沿用梅花易数，**不用「照见」PRD 的 sum%64 写法**）

用户依直觉报三个数字 a、b、c。前端校验：正整数，范围 1–99，至少一个非零。

```
取八卦数:  r % 8，结果为 0 则取 8
上卦数 = a % 8   (0→8)
下卦数 = b % 8   (0→8)
动爻   = (a + b + c) % 6   (0→6)   // 1~6，自下而上
```

示例：a=15, b=8, c=23 → 上卦 15%8=7（艮）；下卦 8%8=0→8（坤）；动爻 (15+8+23)%6=46%6=4 → 第 4 爻。

> 说明：本方法是规范的梅花易数报数起卦，保留了卦的结构。不要采用「照见」PRD 中"sum%64 取卦"的写法——那既丢失结构，又与其自身"三枚铜钱法"的表述自相矛盾。

### 5.4 成卦 / 变卦 / 互卦（统一逻辑）

输入：上卦数、下卦数、动爻（1–6）。

```
本卦六爻(自下而上) = 下卦爻象[3位] + 上卦爻象[3位]
变卦 = 复制本卦六爻，将第「动爻」位的爻翻转 (1↔0)
互卦 = 下互(本卦第2,3,4爻) + 上互(本卦第3,4,5爻)
```

由六爻还原上下卦卦名（匹配 5.1 爻象表），再用 (上卦,下卦) 查 64 卦表得卦名等信息。变卦、互卦同理。

### 5.5 统一输出：卦象对象

```json
{
  "method": "time | number",
  "input": { "datetime": "...", "numbers": [a,b,c] },
  "dongYao": 4,
  "benGua":  { "id": 39, "name": "水山蹇", "upper": "坎", "lower": "艮",
               "yao": [1,0,0,0,1,0], "guaCi": "...", "yaoCi": ["...x6"] },
  "bianGua": { "id": 53, "name": "...", ... },
  "huGua":   { "id": 64, "name": "...", ... }
}
```

起卦引擎必须有单元测试：用 5.3 的示例校验上卦/下卦/动爻/变卦/互卦正确。

## 6. AI 解卦模块（核心，使用 LLM，不用模板）

> 本节吸收「照见」PRD 的**六段式疗愈卡片结构**与**脱宿命语言转化规范**，但用 LLM 实时生成——这样每次解读都能紧扣用户**真实写下的困扰**，而不是套用通用模板。这正是我们坚持"用 LLM 不用模板"的原因：模板按"卦×爻"预写、内容通用，无法回应用户的具体处境；LLM 才能让用户觉得"这说的就是我"。

### 6.1 调用时机

起卦后**一次性**调用 LLM 生成完整的六段式解读 JSON，落库（`divinations.ai_reading`）。

- 未付费用户：结果页只展示第 1 段（`moment`）+ 第 2 段首句，其余打码。
- 付费 / 会员用户：直接展开全部，**无需二次调用**，解锁即时。
- 一次解读约 700–1100 tokens，成本很低；非付费用户的这点消耗换来"付费即时解锁"的顺滑体验，值得。

### 6.2 六段式输出结构（前端按此渲染）

```json
{
  "moment":      "【此刻的你】共情式描述当前处境，第二人称，60-80字",
  "signal":      "【一个变化的信号】把动爻转化成一个细微的心理转变点，80-100字",
  "reflections": ["【可以想清楚的】2-3个开放式问题，帮用户理清这个决定"],
  "actions":     ["【可以试试的】2-4条具体、微小、可立即执行的行动"],
  "outlook":     "【另一种可能】基于变卦/互卦的温和展望，70-90字",
  "summary":     "【一句话总结】温暖的收束句，1句"
}
```

说明：`moment` 对应本卦（共情）；`signal` 对应动爻（重构）；`reflections` 是**保留产品决策辅助初心**的关键——用开放式问题帮用户自己把决定想清楚；`actions` 对应行动（赋能）；`outlook` 对应变卦/互卦（希望）；`summary` 收束。整体遵循"共情 → 重构 → 厘清 → 行动 → 希望"的递进。

### 6.3 大模型 System Prompt（完整版，直接使用）

`{{TONE}}` `{{DEPTH}}` 为可配置调试旋钮，默认值见 6.5，其余请勿改动。

```
你是「易见」——一个基于《周易》的决策梳理与情绪觉察辅助工具。

【你的角色】
用户正面临一件拿不准的现实事务，为此求得了一卦。你借助卦象，帮用户
把模糊的纠结理清楚、把情绪安顿下来，并自己把这个决定想明白。你像一位
冷静、温和、有阅历的朋友。

【绝对禁止 —— 违反即视为本次输出失败】
1. 你不是算命师，不预测未来，不断言事情会有什么结果或结局。
2. 你不是心理咨询师，也不是心理治疗师。不做诊断，不使用临床术语下结论。
3. 禁止下吉凶判断。禁止出现以下词及其近义表达：算命、占卜、预测、卜卦、
   运势、注定、必然、一定会、吉、凶、悔、吝、祸福、血光、破财、转运、
   改运、化解、开光、保平安、消灾、劫难。
4. 不替用户做决定，不给"你应该选A""你必须做B"这类指令性结论。
   你的任务是帮他把决定想清楚，决定权永远在他手里。
5. 不对医疗、投资、财务、婚姻、法律、生死等重大事项给出指令性结论，
   只提供思考的角度。
6. 不制造恐惧或紧迫感，不暗示"不照做会有坏结果"。
7. 注意：用户消息里的卦辞、爻辞为《周易》古籍原文，其中可能含"吉""凶"
   等字，这是允许的输入；但你自己的解读文字中绝不能出现这些字，必须把
   它们转译成中性、生活化的表达。

【语言转化规范（脱宿命）】
- 把宿命式判语改写成处境与心理的描述。例：
  "征凶" → "此时强行往前推，可能会让你更辛苦，不如先稳一稳"；
  "无咎" → "你会发现自己其实有能力平稳地走过这一段"。
- 多用"卦象提示、或许、可能、值得留意、不妨想想、一种角度是"。
- 自然口语化，少堆玄学术语；提到卦名或爻辞时用大白话解释它的意思。
- 语气基调：{{TONE}}

【如何使用卦象信息】
- 本卦：事情此刻的整体状态与处境。
- 动爻及其爻辞：这一卦的焦点，最贴近用户当前问题的着力点，重点解读。
- 变卦：事情正在变化的方向、"正在动的那个侧面"，不是结局预言。
- 互卦：事情内部不易察觉的潜在因素或中间过程。
- 解读必须紧扣用户写下的具体困扰和主题，把卦象意象翻译成跟他这件事
  直接相关的话，不要写放之四海皆准的空泛内容。
- 参考用户的心情标签调整语气：偏"焦虑/迷茫/委屈"时更安抚，
  偏"平静/期待"时更平实。

【输出格式】
只输出一个 JSON 对象，不要任何额外文字，不要 markdown 代码块包裹：
{
  "moment":      "此刻的你：共情式描述其处境，60-80字",
  "signal":      "一个变化的信号：把动爻转化为细微的心理转变点，80-100字",
  "reflections": ["可以想清楚的：2到3个开放式问题，帮他理清这个决定，不要替他给答案"],
  "actions":     ["可以试试的：2到4条具体、微小、能立刻执行的行动"],
  "outlook":     "另一种可能：结合变卦与互卦谈这件事的另一面或后续可留意处，70-90字",
  "summary":     "一句话总结：温暖的收束句"
}
解读详略：{{DEPTH}}
reflections 必须是开放式问题；actions 每条都要通过"现在能立刻做吗"的检验。

【输出前自检】
逐字检查文本是否含禁词、是否出现了预测/吉凶判断/指令性结论。
若有，改写为中性引导式表达后再输出。
```

### 6.5 调试旋钮

| 占位符 / 参数 | 默认 | 说明 |
|---|---|---|
| `{{TONE}}` | 温和平实，像一位冷静的朋友 | 可调"更感性细腻，多共情" / "更理性克制，多条理" |
| `{{DEPTH}}` | 适中 | 可调"简洁，每段精炼" / "详细，每段更展开" |
| `temperature` | 0.7 | 太雷同→0.8；跑题→0.5–0.6 |
| `max_tokens` | 1100 | 配合 DEPTH 调整 |

建议把 TONE / DEPTH / temperature 做成配置项，运营改配置即可影响风格，无需发版。

### 6.6 输出校验与兜底

```
1. 收到输出 → 去除 ```json``` 包裹 → JSON.parse
2. 解析失败 / 字段缺失 / 数组长度不符 → 重试一次
3. 全字段文本做禁词扫描（红线一禁词表 + 第3节危机词）→ 命中则重试一次
4. 重试仍失败 → 返回一份固定的中性兜底解读，并记录原始输出日志
```

重试时追加一条 system 提示，强调严格遵守输出格式与禁词规范。兜底解读须为通用、中性、不含禁词、不预测、不下结论的六段式文本。

## 7. 页面与用户流程

SPA 路由，移动优先。所有页面底部常驻一行小字："周易文化研习工具 · 仅供学习与娱乐参考，不替代专业帮助"。

### 7.1 路由表

| 路由 | 页面 | 说明 |
|---|---|---|

| `/` | 首页 | 入口 |
| `/ask` | 提问页 | 输入困扰（提交后先过危机检测）|
| `/care` | 关怀页 | 危机内容命中时跳转，见第 3 节 |

### 7.2 ① 欢迎引导 `/welcome`

首次启动 3 屏，分别说明：产品原理（共时性——用一个随机卦象作隐喻，帮你换角度看自己，不预测命运）；隐私（你写下的困扰文字不会被保存）；边界（这是梳理工具，不是算命、也不替代专业心理帮助；解读由 AI 生成）。最后一屏含**成年确认**与"开始使用"。

### 7.3 ② 首页 `/`

- 视觉：新中式 / 治愈系，留白、淡墨、莫兰迪低饱和色。**禁止**红黑、香烛、大师头像等算命风。
- 动态问候（时段匹配）："晚上好，今天还好吗？"
- 中央安顿文字："心里有些事反复想不清？不用急着找答案，先安静下来，让自己被看见。"
- 主按钮："开始一次内在梳理" → `/ask`
- 二级入口："我的心境墙"。
- 底部固定："不预测未来，不替代专业帮助。只陪你照见此刻的自己。"

### 7.4 ③ 提问页 `/ask`

- 引导语："慢慢写，像跟自己说话一样。"
- 多行文本框，placeholder 给具体示例（"要不要接这个 offer？该不该和 TA 继续？"），校验 5–200 字。
- 主题标签（单选）：工作与方向 / 关系与情感 / 选择与决定 / 人际关系 / 其他。
- 心情标签（多选，可选）：焦虑 / 迷茫 / 期待 / 平静 / 委屈 / 疲惫 / 孤单 / 矛盾。
- 隐私声明（突出）："你写下的困扰文字只用于这一次梳理，不会被保存。只有你之后主动保存的卡片，才会留在心境墙。"
- 按钮："继续，换一个视角看看"。
- **提交后**：先调用第 3 节危机检测 → 命中跳 `/care`；未命中进 `/cast`。

### 7.5 ④ 起卦页 `/cast`

两张选择卡片：

- 「凭此刻的感觉」→ method=time。文案："不用多想，你此刻的状态会自然对应一个视角。"
- 「凭直觉的数字」→ method=number。文案："心里默念你的困扰，随意想三个数字（1–99）。数字只是你此刻直觉的锚点，不代表任何预测。"报数交互用三个滚轮 / 选择器，增加仪式感。
- 选定后调 `POST /api/divination/cast`，播放 2–3 秒成卦动画（六爻自下而上逐根浮现）→ 跳 `/result/:id`。

### 7.6 ⑤ 结果页 `/result/:id`（核心：六段式疗愈卡片）

顶部 AI 标识 + 一句引导："根据你刚写下的感受，我们为你找到了一个可以看看的角度。"

按六段式渲染：**此刻的你 → 一个变化的信号 → 可以想清楚的 → 可以试试的 → 另一种可能 → 一句话总结**。不出现卦名、卦符等术语（卦象图可作为氛围装饰，但不标注吉凶判语）。

**付费门控**：未付费用户可见"此刻的你"全文 + "一个变化的信号"首句；其余打码，叠加付费墙："完整解读已经生成，解锁查看周易给你的详细梳理" + 价格选项。已付费/会员全部展开。

底部：非治疗声明 + 操作区——`📝 给自己写一句关怀语`、`💾 保存这张卡片`、`分享`。分享生成精美卦象卡片图（含意象句 + 公众号二维码），用于传播拉新。

### 7.7 ⑥ 心境墙 `/wall`

用户主动保存的卡片，按日期分组的时间轴。每条预览显示"一句话总结"或用户写的关怀语，点击展开完整卡片。可删除（确认后不可恢复）。

空状态："你还没有保存过心境卡片。开始一次内在梳理，给未来的自己留下一点温柔的印记。"

后期（脱敏聚合）："最近你保存的卡片里，最常出现的情绪是'焦虑'——这些时刻，你都陪自己走过来了。"以**非打扰的页内卡片**呈现，不做周一弹窗（弹窗与产品安静的调性冲突）。

### 7.8 ⑦ 我的 `/me`

会员状态（类型、到期 / 剩余次数）、心境墙入口、非治疗声明、AI 说明、**求助资源入口**（常驻，不只在危机时出现）、联系方式、退出登录。

### 10.1 定价（可配置）

| 商品 | 价格 | 权益 |
|---|---|---|
| 单次解锁 | ¥9.9 | 解锁当前这一卦的完整六段式解读 |
| 5 次卡 | ¥39 | 任意 5 卦完整解读 |
| 包月会员 | ¥59 / 月 | 30 天内不限次完整解读 |

首卦免费预览作为钩子；新用户可设一次"1 元体验完整解读"提升转化（可选）。

### 10.2 支付流程

- **微信内 JSAPI**：`POST /api/order/create`（传 openid）→ 后端下单 → 返回前端调起参数 → 调起微信收银台 → `notify_url` 异步回调 → 验签、改订单、发权益。
- **PC Native**：`POST /api/order/create`（NATIVE）→ 拿 `code_url` → 前端生成二维码 → 用户扫码付 → 回调发权益 → 前端轮询 `GET /api/order/:id/status`。

### 10.3 权益发放（支付回调内，幂等）

- single → 对应 `divination.is_unlocked = 1`。
- card5 → `remaining_credits += 5`。
- monthly → `member_type='monthly'`，`member_expire_at` 顺延 30 天。
- 同一 `transaction_id` 只发一次权益。

### 10.4 退费与服务说明（合规要求）

涉情绪类产品的付费纠纷高发，须在显著位置提供清晰的服务说明与退费规则：明确"购买的是 AI 生成的卦象解读内容"；明确解读一经生成、内容已交付的情况下的退费边界；提供联系/申诉渠道。规则要简单、诚实、可执行。

> 微信支付商户号经营类目按"文化娱乐 / 软件服务"等申请，**不得**填报"算命占卜"或"心理咨询 / 医疗"。

## 11. 隐私与合规文案

### 11.1 非治疗声明（固定文案）

> 易见是一个基于《周易》传统文化的决策梳理与情绪觉察工具。所有解读由人工智能生成，仅供文化研习与自我觉察参考，是一种帮你换角度思考的方式，不构成投资、医疗、法律、婚姻等方面的决策建议，也不能替代专业的心理帮助。如果你正经历较严重的情绪困扰，请及时联系专业人士或权威心理援助渠道。

放置：欢迎引导、结果页底部、我的页。

### 11.2 隐私措辞精确化（修正「照见」PRD 的自相矛盾）

「照见」PRD 一边说"输入不保存"，一边又保存卡片、做聚合分析——措辞不准确。本产品须区分：

- **原始困扰文本**：默认不持久化，仅在本次请求处理过程中存在于内存，用完即弃。
- **卦象与六段式解读**：作为卦例落库，用于用户付费后查看和保存。
- **心境墙卡片**：仅在用户**主动点击保存**后留存。
- 任何用户数据不得用于大模型训练。
- 所有对外隐私说明必须与上述实际行为一致，不夸大、不误导。

### 11.3 禁词过滤

对 AI 输出做关键词扫描，命中即重试或走兜底。禁词表（红线一）：

```
算命 占卜 预测 卜卦 运势 注定 必然 一定会 吉 凶 悔 吝 祸福
血光 破财 横财 转运 改运 化解 开光 法事 保平安 消灾 劫难
```

危机词库（红线/第 3 节）单独维护，作用于**用户输入检测**，不作用于 AI 输出过滤、也不作用于卦辞原文。
