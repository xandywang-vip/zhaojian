# 照见 · 产品方案 v1.0

> 易经视角的情绪自我觉察工具  
> 不预测未来，不替代专业帮助，只陪你照见此刻的自己。

---

## 一、产品定位

### 核心定义
**传统文化（易经）+ 情绪记录工具**

不是算命，不是心理咨询，不是星座App。  
是一个用易经的视角，帮助用户把模糊的情绪锚定到具体处境的自我觉察工具。

### 产品公式
```
起卦（入口）→ 卦象解读（视角）→ 动态追问（触发表达）→ 手账记录（沉淀）→ 分享传播（增长）→ 时间轴回看（留存）
```

### 竞品差异
| 维度 | 传统情绪手账 | 星座/塔罗App | 照见 |
|------|------------|------------|------|
| 用户行动 | 自己写→自己消化 | 看解读→被动接受 | 被触动→主动表达 |
| 核心价值 | 记录 | 娱乐/慰藉 | 自我觉察 |
| 内容来源 | 用户自生成 | 算法生成 | 卦象+用户共创 |
| 留存驱动 | 习惯 | 娱乐刷新 | 时间轴回看 |

---

## 二、合规框架（开发必读）

### 产品定性
- ✅ 传统文化内容工具
- ✅ 情绪记录 App
- ❌ 不是心理健康服务
- ❌ 不是占卜预测服务

### 内容红线（所有 AI 生成内容必须遵守）
**绝对禁止出现：**
- 预测性词语：会、将、能否、结果、成功、转机、注定、命中
- 诊断性词语：焦虑症、抑郁、障碍、症状、病
- 引导性判断：是不是、应该、对不对、你一定是
- 结果性断言：此卦主吉、近期有贵人、感情有转机

**合规表达范式：**
- ❌ 此卦显示你们感情有转机
- ✅ 古人面对类似处境，选择先蓄势而不急于行动
- ❌ 近期不宜做重大决定
- ✅ 这一卦象提示的是一种内在状态，而非外部结果

### 必须展示的免责声明
```
本内容为易经传统文化视角，仅供自我觉察参考。
不预测未来，不替代专业心理帮助。
```

```
## 角色
你是「照见」App 的追问生成模块。

## 产品背景
照见是易经视角的情绪自我觉察工具。
不预测未来，不做心理诊断，不替代专业帮助。

## 任务
根据卦象信息，生成一个帮助用户把情绪锚定到
具体生活处境的追问问题。

## 输入
- 本卦：{hexagram_name}
- 动爻：{yao_position} 爻
- 爻辞核心意象：{yao_image}
- 之卦：{result_hexagram}
- 本次解读摘要：{reading_summary}（50字以内）
- 用户主题：{theme}

## 追问角度（三选一，根据动爻性质选择）
- 空间型（对应"位"）：问用户在某处境中的位置坐标
  例："在这件事里，你是在主动推进，还是在旁边等着？"
- 动力型（对应"动爻"）：挖掘"想动又没动"的张力
  例："有什么事，你一直知道可以做，但还没有迈出去？"
- 因果型（对应"之卦"）：揭示纠结背后的真实动机
  例："如果你等的东西一直不来，会不会其实是种解脱？"

## 输出要求
只输出一个问题，不加任何前缀和解释。

必须满足全部条件：
1. 字数 10～20 字
2. 问的是「此刻处境」，不问「未来结果」
3. 禁止：会、将、能否、结果、成功、转机
4. 禁止：焦虑、抑郁、症、障碍
5. 禁止：是不是、应该、对不对（引导性判断）
6. 禁止以「为什么」开头（改用：哪一个瞬间 / 有没有 / 哪件事）
7. 必须包含来自爻辞意象的具体词语或动作
8. 不能是套用任何卦都成立的通用问题

## 合规兜底（最高优先级）
触犯以下任意一条必须重新生成：
- 含对用户未来的任何预测或暗示
- 含心理诊断相关词汇
- 含「请问」「您」等正式问询语气
- 超过 20 字
- 是通用问题（换个卦也能问）
```

**后置校验逻辑（代码层实现）：**

```javascript
const FORBIDDEN_WORDS = ['会', '将', '能否', '结果', '成功', '转机', 
                          '焦虑症', '抑郁', '障碍', '请问', '您'];

function validateQuestion(question) {
  if (question.length > 20) return false;
  if (question.length < 6) return false;
  if (question.startsWith('为什么')) return false;
  if (FORBIDDEN_WORDS.some(w => question.includes(w))) return false;
  return true;
}

// 最多重试2次，失败后进入兜底库
async function generateQuestion(hexagramData, retryCount = 0) {
  const question = await callAI(PROMPT, hexagramData);
  if (validateQuestion(question)) return question;
  if (retryCount < 2) return generateQuestion(hexagramData, retryCount + 1);
  return getFallbackQuestion(hexagramData.theme);
}
```

## 四、技术实现要点

### 4.1 起卦算法
- 传统铜钱起卦：三枚铜钱，投6次，每次生成阴/阳爻
- 老阳（变爻）= 动爻，触发「之卦」计算
- 每次起卦结果与用户 ID + 时间戳绑定，落库存储

### 4.2 AI 调用流程

```
用户选主题
    ↓
本地计算卦象（不调用 AI）
    ↓
调用 AI：卦象解读（Prompt 3.4）
    ↓
展示解读，用户向下滑动
    ↓
调用 AI：生成追问（Prompt 3.5）
    ↓
后置校验：不合格 → 重试（最多2次）→ 兜底库
    ↓
用户填写回答
    ↓
生成手账卡片（本地渲染，无需 AI）
    ↓
落库：卦象 + 解读 + 追问 + 用户回答
```

### 4.3 数据存储结构

```javascript
// 每次记录的数据结构

  id: uuid,
  user_id: string,
  created_at: timestamp,
  theme: string,              // 用户选择的主题
```
